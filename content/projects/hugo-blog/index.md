这是一个使用 Hugo 和 Blowfish 构建、由我独立设计全栈搭建维护并持续更新工程化功能的个人科研/技术数字花园。

## 1. Engineering Problem

**这个项目解决什么问题？为什么值得研究？**
作为深耕在实体工程和数据底层的工程师，经常遇到的痛点是传统的笔记软件数据全锁定或者排版僵化，而传统的第三方平台要么充斥广告，要么不开源限制长图与深层内容展示。因此自主设计并维护一套分离强静态高速网页端与活态动态后端数据的架构系统不仅是为了“做笔记归类”，更是一次探索前沿静态网页整合 Serverless 数据库、进行全权限分离与自定义 UI 展示工程的绝佳靶场。

## 2. My Role

**我具体负责什么？**
- 搭建基础核心系统拓扑，挑选轻量高性能静态引擎渲染管道完成从无到有建站。
- 系统架构级别的 UI 交互层设计实现逻辑更新（CSS 封装重定义与 JavaScript 原生轻量 DOM 控制）。
- 部署并在平台完整实施全系统开源到部署闭环流水线 DevOps（基于 Git）。
- 在服务端以无服务器模式完成 Supabase 后端接口服务深度聚合集成与关系表结构搭建验证。

## 3. Method

- **静态核心处理**：用 Go 语言编写编译的 Hugo 引擎完成 markdown 预编译，结合功能模块化结构化强大的前沿极简 Blowfish 主题。
- **全栈数据支持**：集成对接 Supabase (PostgreSQL 数据库)，保存讨论、留言等所有富含 UGC 原生状态记录的表数据。
- **核心身份模型与鉴权机制**：底层设计通过严格遵守 Supabase 强鉴权框架，撰写及开启实施 RLS(Row Level Security)，配置验证端（Admin / Authorized User / Public），建立起完备的隔离防击穿权限层网。
- **自动化流**：打通 Github Actions 远端钩子、直接投射推向自运营云 GitHub Pages 解析节点以承接海量不间断流量并发。

## 4. Key Results

- 大幅重整杂乱科研散记，梳理重构成现在涵盖了数理推导分享支持并涵盖全文超速秒搜检索的清晰索引版块，做到了极致 SEO 和知识复盘化。
- 实现交互社区动态。数据库交互全请求开销耗时控制极低在超百毫秒内加载填充完毕且未建立甚至不用支出购买繁冗中间机。
- 将原本被归档作为记录的“玩具项目页”彻底整改为专业科研工程“强证明力展示页”（如本文档的层级剖面展示分析能力），直接反映了逻辑重构的方法论落地。

## 6. What I Learned

**遇到的问题与解决思路：**
- **跨前端域的访问与安全性界限突破**：原本单薄只借助前端 JavaScript 来隐层特定非公开的内容页节点实际上形同裸露接口可以直接扒取内容。**失败方案**：尝试基于 JS Token 二重混淆均易被劫持。**解决**：重组理解底层结构从应用切到数据驱动端切入行级权限限制指令（RLS），真正做到了无法获取直接返回无内容，大幅填补了身份安全。
- **过重动效引起渲染抖动和拖影**：起步时过度附加 JavaScript DOM 位控实现悬浮效果，反而让主流低配置机型发热耗电及卡死。**解决**：进行性能压测后对重排大户组件实施样式剥离，全线过渡更改为了原生物理引擎支持的 GPU 层 `transform/opacity/will-change` 流畅层操作组合。
- **下一步优化**：利用现有完善系统对接扩展接口支持更多数理图表交互 API 模块化展示分析，深度充沛工程内容的图文配比使得名片更加具有冲击力和不可替代属性。
